# 🧭 dev-agents-team

### Multi-Agent Development Team

> **9个角色 × 3档任务模式 = 覆盖 coding、Skill 开发与项目建设的完整流水线**

> **v2.0 升级版**：参考 GitHub agency-agents + 顶级开发工程实践（Google SRE / ThoughtWorks 技术雷达 / Martin Fowler 架构演化），每个角色从「能用」升级为「专家级开发搭档」。

## 🎯 任务模式

### 🟢 轻量模式（3人）

适用：单文件脚本、Skill 微调、临时小改动

```
用户 → 司南 → 工程师（铸铁/织网/刻石三选一）→ 白泽 → 青龙 → 司南 → 用户
```

### 🟡 标准模式（5–6人）

适用：有前后端的功能模块、完整 Skill 开发、小型项目

```
用户 → 司南 → 罗盘 → 青图 → 工程师（并行）→ 白泽 → 青龙 → 司南 → 用户
```

### 🔴 完整模式（9人）

适用：完整项目、多模块并行、严肃质量要求。御史全程开启。

## ⚙️ 工作流程

```
1.  用户 → 司南: 提交需求
2.  司南: 判断任务类型，宣布启用模式（TDS 评分）
3.  罗盘: 需求分析 → 输出需求规格文档（含 NFR）→ 司南
4.  青图: 架构设计 → 输出技术方案（含性能预算 + 技术雷达）→ 司南
5.  司南: 拍板方案，拆分任务，分配给织网/铸铁/刻石
6.  工程师（并行）: 实现 → 交付白泽审查
7.  白泽: 审查（代码味道四维分类）→ 打回修改（工程师重交）/ 通过（司南终审）
8.  司南终审 → 通知青龙准备交付
9.  青龙: 环境/部署/SLO 检查 → 向司南汇报
10. 司南 → 用户: 汇报交付内容，请求确认
11. 用户确认 → 司南通知青龙正式发布
12. 御史: 全程监察（指标级联 + 效率基线），发现问题随时上报司南
```

**自动衔接，司南不介入：**

- 罗盘 ↔ 用户（追问需求）：自动衔接
- 工程师 ↔ 白泽（修改重审循环）：自动衔接

## 📥 安装

每个 Agent 对应一个目录，内含一个 `SKILL.md` 文件。

将 `agents/` 目录复制到你的 AI 平台的 Skills 目录：

```
# 用户级（所有项目通用）
~/.skills/

# 项目级
<你的项目路径>/.skills/
```

安装后在技能面板中激活 `司南-技术总监` 即可启动团队。

---

## 🚀 快速开始

1. 将 `agents/` 目录放入 AI 平台的 Skills 目录
2. 在技能/专家面板中选择 `司南-技术总监`
3. 发送你的开发需求

司南会自动判断任务类型，宣布启用模式并启动对应团队。
